## 一、项目概述

### 1.1 项目背景

在数字化与智能化深度融合的时代，高等教育的个性化变革成为核心发展方向。不同学生在知识基础、学习能力、兴趣方向上的显著差异，使得标准化教学难以满足个性化学习需求。本项目旨在借助大模型技术体系（通用大模型、多模态生成、AI 辅助编程工具等），构建高等教育个性化学习资源体系，开发智能学习智能体系统，实现"因材施教"的数字化落地。

### 1.2 项目目标

以**"人工智能导论"**课程为切入点，构建多智能体系统，实现：

1. **对话式学习画像自主构建**：通过自然语言对话自动提取≥6 维度的学生画像，支持动态更新
2. **多智能体协同资源生成**：多个 AI Agent 协作生成文档、思维导图、练习题、视频脚本、代码示例等 5 类学习资源
3. **个性化学习路径规划**：基于知识图谱和学生画像，自动生成拓扑排序的个性化学习路径
4. **智能辅导**：基于 RAG 检索增强的多模态答疑，支持来源引用和难度自适应
5. **学习效果评估**：五维雷达图 + AI 评估报告 + 趋势分析

### 1.3 系统特色

| 特色 | 说明 |
|------|------|
| 多智能体协作 | 4 个 AI Agent 协同工作（RAG 检索→需求分析→专业生成→安全审查） |
| 六维动态画像 | 知识基础、认知风格、易错点、学习目标、可用时间、兴趣方向 |
| 知识图谱驱动 | 10 章 DAG 拓扑排序 + 画像权重个性化排序 |
| SSE 流式输出 | 所有 AI 生成接口均支持 Server-Sent Events 流式推送 |
| 多模态支持 | 文本 + LaTeX 数学公式 + Mermaid 图表 + 代码高亮 + 数字人 + 语音 |
| 双层安全审查 | 正则快筛 + LLM-as-Judge 深度审查 |

| 痛点 | 具体描述 | 解决方案 |

| 学习资源繁杂无序 | 网络上 AI 学习资料海量但质量参差不齐，难以筛选 | 多智能体协同生成高质量、结构化学习资源 |

### 3.2 数据流设计

#### 3.2.1 对话画像流（Chat → Profile）

```
用户输入消息
  → FastAPI 接收 (chat.py)
  → 加载对话历史 + 学生画像 (DB)
  → RAG 检索相关知识 (ChromaDB + Spark KB)
  → 实时学习状态分析 (realtime_state_service)
  → 构建系统提示词 (prompts.py → chat_system())
  → LLM 流式生成 (spark_service.chat_stream())
  → SSE 推送文本到前端
  → 响应结束后: ProfileCoordinator 提取画像 → 写入 DB → SSE 推送画像更新
  → 保存对话消息到 DB
```

#### 3.2.2 资源生成流（Multi-Agent Pipeline）

```
用户输入主题 + 选择类型
  → ResourceCoordinator 编排 4 步 Pipeline:
    Step 1: RAG 检索 (课程知识库 + 用户文档 + Spark KB + 联网搜索)
    Step 2: Orchestrator Agent 需求分析 + 资源规划
    Step 3: 专业 Agent 内容生成 (doc/mindmap/quiz/video/code)
    Step 4: Safety Agent 双层安全审查
  → 每步通过 SSE 推送 agent_status 事件 (working/done/error)
  → 最终内容通过 SSE text 事件流式输出
  → 前端 Agent 状态面板实时显示各 Agent 工作状态
```

#### 3.2.3 学习路径生成流

```
用户请求生成路径
  → 加载学生画像 + 知识图谱 (10 章 DAG)
  → LLM 生成个性化路径结构 (spark_service.chat())
  → Kahn 拓扑排序 + 画像权重调整
  → SSE 推送路径数据 (path_data) + 分析文本 (text)
  → 前端渲染: 时间线视图 / 交互式知识图谱 (SVG)
```

## 六、多智能体系统设计

### 6.1 智能体架构总览

系统设计了 **13 个 AI Agent**，分布在 4 个协作链路中：

```
┌──────────────────────────────────────────────────────────────┐
│                    多智能体协作架构                             │
│                                                              │
│  ┌─ 资源生成链路 (ResourceCoordinator) ─────────────────┐    │
│  │  RAG 检索助手 → Orchestrator → 专业 Agent → 安全审查  │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │
│  ┌─ 画像提取链路 (ProfileCoordinator) ──────────────────┐    │
│  │  ProfileExtractor → 画像写入 DB → SSE 推送更新        │    │
```

#### 自适应难度计算


|------|------|

### 6.3 画像提取编排器（ProfileCoordinator）

**核心文件**：`agents/profile_coordinator.py`（93 行）

#### 提取流程

1. 取最近 6 条对话消息
2. 格式化为"学生: ... / AI: ..."对话文本
3. 包含现有画像作为增量更新上下文
4. 调用 LLM 提取结构化 JSON（temperature=0.1，确保稳定性）
5. 鲁棒性 JSON 解析（处理 Markdown 代码块、多余文本等）
6. 返回 7 字段画像字典

#### 动态节流策略

| 画像完整度 | 提取频率 | 理由 |
|-----------|---------|------|
| 画像不完整（< 3 维有值） | 每轮对话提取 | 尽快构建完整画像 |
| 画像基本完整（3-5 维有值） | 每 3 轮提取一次 | 平衡精度与效率 |
| 画像完整（6 维全有值） | 每 5 轮提取一次 | 避免不必要的 LLM 调用 |

所有 Agent 遵循统一规范：

- **命名**：以"你是智学 [角色名]"开头
- **目标**：明确 Agent 的核心任务
- **输出格式**：指定输出结构（Markdown / JSON）
- **约束条件**：安全规则、幻觉防范规则

### 8.2 核心服务详解

#### 8.2.1 SparkService（LLM 客户端）

- **接入方式**：讯飞星火大模型 REST API
- **模型**：spark-x
- **核心方法**：
  - `chat(messages, temperature, max_tokens)` → 同步完整响应
  - `chat_stream(messages, temperature, max_tokens)` → 异步 SSE 流式响应
- **超时设置**：120 秒
- **错误处理**：自动重试 + 异常传播

#### 8.2.2 RAGService（RAG 检索服务）

- **向量库**：ChromaDB（PersistentClient，路径 `data/chroma/`）
- **核心集合**：
  - `ai_intro_course`：课程知识库（10 章 Markdown，按 `##` 标题分块）
  - 用户文档集合：按用户隔离，支持增量更新
- **检索策略**：
  - 语义向量检索（余弦相似度）
  - 文本模糊匹配兜底（向量检索失败时）
  - 多源合并去重（课程 KB + 用户文档 + Spark 云端 KB）
- **自动修复**：检测到嵌入维度不匹配时自动重建集合

#### 8.2.3 SafetyService（安全审查服务）

**第一层 — 正则快筛（10 类敏感模式）：**

| 编号 | 类别 | 示例模式 |
|------|------|----------|
| 1 | 暴力内容 | 杀、打死、伤害 |
| 2 | 武器 | 枪支、炸药、刀具制作 |
| 3 | 犯罪 | 偷盗、诈骗、贩毒 |
| 4 | 自残 | 自杀、割腕、跳楼 |
| 5 | 歧视 | 歧视、侮辱、种族 |
| 6 | 色情 | 色情、裸露 |
| 7 | 政治敏感 | 政治相关词汇 |
| 8 | 个人隐私 | 身份证号、手机号 |
| 9 | 医疗误导 | 诊断、处方 |
| 10 | 金融诈骗 | 投资回报、理财骗局 |

**附加检测：**
- 幻觉标记检测（LLM 常见含糊表述）
- 外部图片 URL 检测（阻止非相对路径、非 data URI 的图片链接）

**第二层 — LLM-as-Judge：**
- `llm_safety_check()`：内容安全性审查（fail-closed，出错默认不安全）
- `llm_hallucination_check()`：幻觉检测（对比上下文，输出置信度）

#### 8.2.4 EmbeddingService（向量化服务）

- **主方案**：讯飞星火 Embedding API（embedding-2，1024 维）
- **降级方案**：本地 n-gram 哈希向量化（当 API 不可用时）
- **自动检测**：集合嵌入方法与当前不一致时自动重建

## 十二、核心算法与服务

### 12.1 SM-2 间隔重复算法

用于闪卡复习调度，基于遗忘曲线原理：

```python
# SM-2 算法核心参数
ease_factor = max(1.3, ease_factor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)))
# quality: 0-5（0=完全忘记，5=完美回忆）
# interval_days: 间隔天数
# next_review_at: 下次复习时间
```

复习调度规则：

| 回忆质量 | 效果 |
|----------|------|
| quality ≥ 3（记住） | 间隔增大，ease_factor 增加 |
| quality < 3（忘记） | 间隔重置为 1 天，ease_factor 降低 |

### 12.3 实时学习状态分析

**核心文件**：`backend/services/realtime_state_service.py`（236 行）

双层分析策略：

| 层级 | 方式 | 说明 |
|------|------|------|
| 第一层 | 规则匹配 | 基于关键词的情绪/困惑检测（无 LLM 调用，< 10ms） |
| 第二层 | LLM 增强 | 复杂语义分析（按需触发） |

5 维实时状态：

| 维度 | 取值范围 | 说明 |
|------|----------|------|
| emotion | positive/neutral/negative | 情绪状态 |
| confusion | 0-1 | 困惑程度 |
| cognitive_load | low/medium/high | 认知负荷 |
| confidence | 0-1 | 自信程度 |
| engagement | low/medium/high | 参与度 |

### 12.4 薄弱点诊断

**核心文件**：`backend/services/diagnostic_service.py`（223 行）

诊断逻辑：
1. 收集用户做题记录（章节 + 正误率）
2. 错误率 > 50% 的章节标记为薄弱点
3. 结合概念本体检查前置概念掌握度
4. 自动更新学生画像的 `weak_points`
5. LLM 生成诊断报告（错误类型分类：概念混淆 / 知识空白 / 粗心错误 / 推理偏差）

### 12.5 每日学习计划

**核心文件**：`backend/services/daily_plan_service.py`（332 行）

计划生成考虑因素：
- 学生画像（知识基础、学习目标、可用时间）
- 当前学习路径进度
- 待复习闪卡数量
- 薄弱点优先级
- 概念本体前置依赖

### 12.6 概念掌握度追踪

**核心文件**：`backend/services/mastery_service.py`（268 行）

基于遗忘曲线的掌握度衰减模型：

```
掌握度随时间衰减：mastery(t) = mastery_0 * e^(-λ * t)
λ = forgetting_factor（因人而异，根据复习频率调整）
每次复习后 mastery 更新并重置衰减起点
```

### 16.2 技术创新点

1. **多智能体协同编排**：ResourceCoordinator 实现 4 步 Pipeline（RAG→Orchestrator→Agent→Safety），支持单阶段和两阶段生成模式
2. **六维动态画像**：基于 LLM 的对话式画像提取，配合动态节流策略和增量更新
3. **知识图谱 + 画像权重**：Kahn 拓扑排序确保前置依赖，画像权重实现个性化排序
4. **双层安全审查**：正则快筛（< 10ms）+ LLM-as-Judge 深度审查，fail-closed 设计
5. **概念本体驱动**：30+ AI 概念的形式化知识体系，支撑路径规划和薄弱点诊断
6. **实时学习状态分析**：5 维实时状态追踪（情绪、困惑、认知负荷、自信、参与度）
7. **SM-2 间隔重复**：基于遗忘曲线的智能复习调度
