# 系统需求分析

## 1. 背景与目标

在数字化与智能化深度融合的时代，高等教育的个性化变革已从可选走向刚需。传统课堂教学无法兼顾每位学生的学习节奏与特点，海量学习资源难以精准匹配个体需求。本项目旨在借助大模型技术和多智能体协同架构，构建高等教育个性化学习资源体系。

**目标用户**：高校学生，以"人工智能导论"课程为切入点。

## 2. 大学生学习痛点分析

| 痛点 | 表现 | 技术解决方向 |
|------|------|-------------|
| 资源杂乱无序 | 海量课程资料、学术文献难以筛选 | RAG + 智能检索，精准匹配 |
| 学习缺乏个性化 | 统一教学进度无法适配个体差异 | 6维画像 + 动态学习路径 |
| 知识薄弱点难定位 | 不知道自己哪里不会 | 学习行为追踪 + 评估诊断 |
| 缺乏即时辅导 | 遇到问题无人解答 | I智能辅导 Agent + RAG 增强 |
| 学习动力不足 | 缺乏学习反馈和成就感 | 可视化进度 + 评估报告 |

## 3. 功能需求

### 3.1 对话式学习画像 (P0)
- 自然语言对话替代表单
- 6个维度：知识基础、认知风格、易错点、学习目标、可用时间、兴趣方向
- 动态更新，随学随新

### 3.2 多智能体协同资源生成 (P0)
- 5种资源：课程文档、思维导图、练习题、视频脚本、实操案例
- CrewAI 编排多智能体协作
- 个性化生成，适配学生画像

### 3.3 学习路径规划与资源推送 (P0)
- 基于知识图谱拓扑排序
- 画像驱动的路径调整
- 多类型资源精准推送

### 3.4 智能辅导 (P1) 加分项
- 多模态答疑（文字 + 图解 + 短视频）
- RAG 增强的课程问题解答

### 3.5 学习效果评估 (P1) 加分项
- 行为追踪与数据采集
- 多维度评估报告
- 闭环优化学习方案

## 4. 非功能需求

- 流式输出避免白屏等待
- Markdown + LaTeX + 代码高亮渲染
- 内容安全过滤 + 防幻觉机制
- 简洁克制 UI 风格
- 基础响应式适配
