# 学径 LearnPath 借鉴分析

> 基于同赛题（第十五届中国软件杯 A3）竞品调研，重点分析 LangGraph 多智能体 + 多通道大模型方案
>
> 对比对象：智学（本系统）— 基于自定义 Coordinator + 星火大模型
>
> 分析日期：2026-06-23

---

## 一、核心架构差异

| 维度 | 智学（我们） | 学径 LearnPath | 借鉴价值 |
|------|------------|----------------|----------|
| **Agent 编排** | 自定义硬编码 Pipeline（RAG→Orch→Agent→Safety） | **LangGraph Supervisor Graph**（意图路由 + 专项节点） | ⭐⭐⭐ 引入意图路由 |
| **学生建模** | 6 维长期画像 | **长期画像 + 实时学情** 双层 | ⭐⭐⭐ 实时学情是最大亮点 |
| **个性化策略** | 少量规则散落各处 | **PersonalizationStrategyService** 规则决策树 | ⭐⭐⭐ 集中策略层 |
| **资源类型** | 5 类（doc/mindmap/quiz/video/code） | **9 类**（+reading/media/ppt/design/project） | ⭐⭐ 扩展类型 |
| **对话 LLM** | 星火（单通道） | Kimi/星火/辅助/Mock 多通道降级 | ⭐ 多通道更可靠 |
| **前端框架** | React + Vite + Tailwind | **Next.js 14** + Ant Design 5 | 各有优劣 |
| **路径规划** | Kahn 拓扑排序 | **LLM 层次化路径** + 异步重规划 Job | ⭐⭐⭐ 异步+层次化 |
| **AI 课堂** | 无 | **完整教学系统**（幻灯片/讲义/测验/PPTX） | ⭐⭐⭐ 最大差异化 |
| **Prompt 管理** | 分散在各 Agent | **`prompts.py`** 集中管理 + 深/浅双模式 | ⭐⭐ 集中管理 |

## 二、最大亮点：实时学情（Realtime Learning State）

### 2.1 概念

学径将「学生是谁」拆成**两个时间尺度**：

| 维度 | 长期画像（Profile） | 实时学情（Realtime） |
|------|-------------------|---------------------|
| 回答 | 你是谁、基础怎样 | 你现在卡在哪、累不累 |
| 更新 | 多轮对话 + 行为刷新 | **每条消息都分析** |
| 字段 | 知识/风格/薄弱点/目标等 7 维 | 情绪/困惑度/投入度/负荷/卡点/好奇点等 |
| 消费方 | 路径规划、资源推荐 | **对话语气、课堂讲法** |

### 2.2 实现方式（`realtime_state_service.py`）

每条用户消息后执行**双模式分析**：

**模式 A — 规则分析（始终运行）：**
- 情绪检测：~30 关键词（"崩溃"→frustrated, "不懂"→confused）
- 困惑度：基于情绪 + 问号数量 + 重复卡点信号
- 认知负荷："抽象/公式/推导/太多/复杂" 等关键词
- 信心："懂了/会了" vs "完全不会/懵"
- 语言风格偏好："简单/通俗/例子/代码/图"

**模式 B — LLM 增强（深度思考模式开启时）：**
- 将规则分析结果 + 最近 12 条消息发给 LLM
- 用 `realtime_state_analysis_system` prompt 做增强分析
- 温度 0.2

**持久化：** 写入 `RealtimeLearningState` 表，供课堂/重规划/对话策略使用

### 2.3 借鉴要点

> 智学可以引入**轻量级实时学情**，在每条对话后做关键词分析（不需 LLM），为辅导和资源推荐增加「当下状态」维度。

## 三、核心差异化：个性化策略层（Personalization Strategy）

### 3.1 架构位置

```
感知层（实时学情）→ 策略层（PersonalizationStrategy）→ 生成层（LLM）
```

### 3.2 规则决策树（`personalization_strategy_service.py`）

输入：`(长期画像, 实时学情, 问题类型)` → 输出完整教学策略：

| 策略维度 | 可选值 |
|---------|--------|
| `teaching_mode` | unblock / simplify / explore / challenge / focus / stabilize / routine |
| `tone` | "温和、清晰、鼓励" |
| `difficulty` | lower / maintain / raise |
| `pacing` | slow / normal / fast |
| `explanation_depth` | brief / standard / deep |
| `response_plan` | 有序行为指令列表 |
| `must_do` / `avoid` | 硬约束 / 禁止项 |
| `preferred_resource_types` | 偏好资源类型 |
| `assessment_style` | 检查方式 |

**教学模式选择逻辑：**

| 条件 | 模式 |
|------|------|
| frustrated/confused, confusion ≥ 0.65 | unblock（疏通卡点） |
| anxious | stabilize（稳定情绪） |
| tired, low engagement | focus（聚焦核心） |
| high cognitive load | simplify（简化） |
| excited, curiosity ≥ 0.7 | explore（拓展探索） |
| high engagement | challenge（挑战提高） |
| 默认 | routine（常规） |

### 3.3 如何注入 LLM

策略**不直接给用户看**，而是格式化为隐藏 prompt 块注入 system prompt：

```
[内部教学策略 — 仅用于调整回答方式，禁止向用户展示]
教学模式：疏通卡点
语气：温和、清晰、鼓励
难度：降低
节奏：慢速
必须做：先确认卡点位置，用类比解释
避免：同时引入新概念
...
```

### 3.4 借鉴要点

> 智学可以引入**集中策略层**，把散落在各处的难度/风格规则统一管理。无需 LLM 参与，纯规则引擎，轻量可维护。

## 四、AI 课堂（最大产品化差异）

### 4.1 设计

每个路径步骤可生成**独立 AI 课堂**，包含：

| 产出 | 说明 |
|------|------|
| `slides` | 8–18 页幻灯片（带 layout/visual_blocks） |
| `handout` | 6–10 节课后复习讲义 |
| `teacher_scripts` | 5 套讲稿（normal/confused/slow/example/practice） |
| `check_question` | 当堂检查题 |
| `mini_quizzes` | 课中小测验 |
| `homework` | 3–6 个课后任务 |
| `image_url` | AI 配图（千问/豆包生成，失败不阻断） |

### 4.2 教学流程

```
导入 → 核心讲解 → 例子演示 → 课中小测 → 反馈纠错 → 课堂总结
```

课中 5 个互动按钮：「听不懂」「讲慢点」「换个例子」「来道题」「已掌握」，每个按钮切换 `teacher_scripts` 对应模式。

### 4.3 与画像的联动

课堂生成时输入包含：**长期画像 + 实时学情 + 教学策略 + 路径节点上下文 + 关联资源**，因此：
- 同一路径步骤，不同时间进入课堂 → **讲法不同**
- 如果刚在对话里表达了「听不懂」→ 新课堂会更慢、更多例子

### 4.4 借鉴要点

> AI 课堂是学径最大的差异化功能，但实现**体量较大**（classroom_service.py 1594 行）。智学可考虑先实现**精简版**：路径步骤绑定教学卡片，用 SSE 生成结构化讲稿 + 随堂测。

## 五、Path 重规划 Job 模式

### 5.1 异步 Job 架构

```
用户提交重规划 → 后台 Job（6 步流水线）→ 前端轮询进度
```

| 步骤 | 内容 |
|------|------|
| 0 | 清除旧路径 |
| 1 | 清除未收藏资源 |
| 2 | 综合分析画像（长期+实时+行为→分析报告） |
| 3 | 重新规划路线（PathAgent 双轮质检） |
| 4 | 重新生成配套资源（按阶段逐个类型） |
| 5 | 最终确认 + 修复断链 |

### 5.2 重规划依据（四层拦截）

| 层级 | 内容 | 来源 |
|------|------|------|
| L1 意图 | 学习目标、对话诉求 | 会话消息、画像 |
| L2 锚点 | 收藏资源、选定资料库 | 用户行为 |
| L3 行为 | 浏览/完成次数、测验记录 | 埋点数据 |
| 分析层 | 学习者综合分析报告 | LLM 生成 |

### 5.3 借鉴要点

> 智学的路径生成是同步 SSE 一次性生成。可引入**异步 Job 模式**，支持：后台重规划不阻塞 UI、进度回显、跨页面浮窗跟踪。

## 六、LangGraph 多 Agent 编排

### 6.1 图结构

```
START → supervisor → profile → END
                   → generate → ... → END
                   → path → END
                   → chat → END
                   → tutor → END
                   → eval → END
```

### 6.2 Supervisor 意图路由

**关键词优先**（非 LLM）：
- `CHAT_META_KEYWORDS` = ["你好", "hi", "help"]
- `GENERATE_KEYWORDS` = ["生成", "创建", "出题"]
- `PATH_KEYWORDS` = ["路径", "规划", "路线"]
- 根据最后一条消息匹配 → 路由到对应节点

**LLM 兜底**：关键词未命中时，用 `INTENT_CLASSIFY_SYSTEM` prompt 分类

### 6.3 借鉴要点

> 智学的 Agent 编排是硬编码 pipeline。可参考 LangGraph 的**意图路由**模式，让用户自然语言直接触发不同 Agent 流程。

## 七、Prompt 设计规范

### 7.1 集中管理（`prompts.py`）

所有 System Prompt 集中在单一文件（1367 行），关键模式：

- **工厂函数**：`*_system(deep: bool = False)` 支持深/浅模式
- **Persona 统一**：全部以「你是学径 LearnPath [角色]」开头
- **JSON 约束**：画像/分析类 prompt 强制"仅输出 JSON"
- **内部策略隐藏**：所有 prompt 包含"禁止向用户暴露内部策略"
- **统一风格**：`CHAT_UNIFIED_REPLY_APPEND` 保证全站语气一致

### 7.2 深/浅双模式

```python
DEEP_THINKING_APPEND = "先分析再回答，800-1500 字..."
FAST_MODE_APPEND = "直接回答，250-450 字..."

def chat_system(deep=False):
    base = "你是..."
    return base + (DEEP_THINKING_APPEND if deep else FAST_MODE_APPEND)
```

### 7.3 借鉴要点

> 智学的 prompt 分散在各 Agent 文件中。可考虑引入**集中 prompt 管理 + 深/浅双模式**。

## 八、前端产品化细节

### 8.1 侧栏 = 五步学习旅程

侧栏导航设计为**编号闭环旅程**：
```
① 智能对话 → ② 学习画像 → ③ 学习路径 → ④ 资源库 → ⑤ 学习评估
```
每个步骤有独立品牌色（蓝/紫/青/橙/绿），顶部有 SVG 进度环。

### 8.2 Keep-Alive 页面架构

所有 7 个页面**同时挂载**，切换仅改 CSS `opacity`，无白屏。登录后预加载全部页面 + 预热 ECharts。

### 8.3 浮窗进度面板

后台异步任务（路径重规划、课堂生成、资源生成）以**可拖拽浮窗**形式呈现，跨页面持续可见，支持最小化/隐藏。

### 8.4 每日最小任务

侧栏 + 路径页展示当日 3 项最小任务（学 1 步路径 + 复习 1 项 + 做 1 道题），降低决策成本。

### 8.5 借鉴要点

> 智学可参考侧栏旅程化和 Keep-Alive 架构，但需注意项目当前是 Vite + React Router，架构调整成本较高。**浮窗进度面板**较易实现，可优先采纳。

## 十、总结

学径 LearnPath 相比智学的**核心优势**：

1. **实时学情** — 每条消息感知学生当下状态，这是最具 AI 味的功能
2. **个性化策略层** — 纯规则决策树，轻量但效果显著
3. **AI 课堂** — 路径 + 课堂的闭环，把"规划"落到"可讲授"
4. **LangGraph 编排** — 意图路由比硬编码更灵活
5. **异步 Job 模式** — 长任务不阻塞 UI，产品体验好
6. **Prompt 工程** — 集中管理 + 双模式 + 风格统一

智学的**核心优势**：

1. **前端 UI 更精致**（Tailwind 手写 vs Ant Design 模板感）
2. **SVG 知识图谱 DAG**（交互式 SVG vs Next.js 卡片列表）
3. **流式体验更完整**（全功能 SSE + Agent 状态面板）
4. **数字人/PPT/语音**等附加功能更丰富
5. **轻量架构**（无 Next.js SSR 复杂性，部署简单）

**最值得优先引入的三件事：**

> 1. **实时学情关键词分析** — 规则引擎，无需 LLM，立即可见效果
> 2. **PersonalizationStrategy 规则引擎** — 统一教学模式选择
> 3. **Prompt 集中管理** — 减少重复维护

> 这三项都不需要大改现有架构，可以在现有系统上增量引入。
