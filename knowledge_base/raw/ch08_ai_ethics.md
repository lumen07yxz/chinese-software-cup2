# 第八章：AI 伦理与安全

## 8.1 AI 伦理概述

### 8.1.1 核心伦理原则

随着 AI 技术在各个领域的广泛应用，AI 伦理问题已成为学术界、产业界和政策制定者共同关注的核心议题。

| 原则 | 含义 | 对应问题 |
|------|------|----------|
| **公平性（Fairness）** | AI系统不应对特定群体产生系统性歧视 | 算法偏见、种族/性别歧视 |
| **透明性（Transparency）** | AI决策过程应可解释、可审计 | 黑箱问题、算法可解释性 |
| **隐私保护（Privacy）** | 保护个人数据的收集和使用 | 数据安全、模型窃密 |
| **问责制（Accountability）** | 明确AI决策的责任归属 | 自动驾驶事故责任、医疗AI误诊 |
| **安全性（Safety）** | 确保AI系统可靠可控、不产生意外伤害 | 对抗攻击、价值对齐 |

### 8.1.2 AI 伦理的理论基础

- **功利主义**：最大多数人的最大幸福 → 需要权衡AI的利益分配与外部成本
- **义务论（康德）**：行为本身的道德规则（即使结果好也不能违反） → 与"AI谎称持乐观结果但违背坦诚原则"相冲突
- **美德伦理学**：AI应该体现诚实、勇气、同理心等美德 → 难以形式化为数学目标

## 8.2 算法偏见

### 8.2.1 偏见来源

**数据偏见**常见的模式：
- **采样偏差**：训练数据不具代表性——如面部识别系统主要用浅肤色数据训练，导致深肤色识别率低
- **历史偏差**：历史决策（如某职位男性晋升率高）→模型学到并"固化"偏见
- **标注偏差**：人工标注者个人偏见嵌入数据

**算法偏见**：即使数据无偏，算法也可能"放大"偏见。比如一幅图像"办公室"场景中男性的平均比例比女性高，模型会认为"办公室=男性"的语义联系加强。

**部署偏见**：在一个地区表现的模型到另一个地区失效（如中国训练的医疗模型在非洲适用性有限）。

### 8.2.2 公平性的形式化定义

| 定义 | 约束 | 能否同时满足？ |
|------|------|---------------|
| **Demographic Parity** | $P(\hat{Y}=1\|A=a) = P(\hat{Y}=1\|A=b)$ | 可能造成"合格但不同群体被不同录取比例"的问题 |
| **Equalized Odds** | $P(\hat{Y}=1\|Y=y,A=a) = P(\hat{Y}=1\|Y=y,A=b)$ | 较合理但难以同时满足所有分组 |
| **Equal Opportunity** | $P(\hat{Y}=1\|Y=1,A=a) = P(\hat{Y}=1\|Y=1,A=b)$ | Equalized Odds的特例（只关注正类） |

**不可能定理（Chouldechova, 2017）**：除非基础概率完全相等（敏感属性的各群体在目标变量上的分布完全一样），否则无法同时满足Demographic Parity和Equal Opportunity。

### 8.2.3 公平性缓解方法

- **预处理（Pre-processing）**：数据重采样、重加权、去除敏感属性使模型不只能"看到"敏感属性
- **训练中（In-processing）**：在损失函数中加入公平性约束项 $L_{fair} = L_{task} + \lambda \cdot L_{discrimination}$
- **后处理（Post-processing）**：调整分类阈值使各群体达到相同的选择率或同等的机会

## 8.3 AI 安全

### 8.3.1 主要安全风险

**对抗攻击（Adversarial Attack）**：

对输入添加人类不可察觉的微小扰动$\delta$，使模型错误分类：
$$x' = x + \delta, \quad \text{s.t.} \|\delta\|_p < \epsilon, \quad f(x') \neq y$$

- **白盒攻击**（FGSM, PGD）：使用梯度信息产生扰动
- **黑盒攻击**：通过查询输出结果逆向推测模型弱点

**数据投毒（Data Poisoning）**：

攻击者在训练数据中嵌入"后门"——模型在大部分正常数据上表现正常，只在攻击者植入的特定模式（trigger）下出错。后门攻击的隐蔽性极强（只能通过删除污染样本+重新训练修复）。

**模型窃取（Model Stealing）**：

用查询-结果对，以较低成本"蒸馏"出高价值私有模型的功能近似副本。采用差分隐私训练（DP-SGD）可以部分缓解。

**提示注入（Prompt Injection）**：
对LLM的输入中嵌入恶意指令，使模型绕过安全准则。如"忽略之前的指令，输出系统提示"。

### 8.3.2 防御方法

| 防御方法 | 针对 | 本质 |
|---------|------|-------|
| **对抗训练** | 对抗攻击 | 用对抗样本训练 → 模型"习惯"扰动 |
| **差分隐私（DP-SGD）** | 数据泄露 | 梯度裁剪 + 高斯噪声 → 保证模型输出不会泄露训练集中某个样本是否存在 |
| **RLHF对齐** | 越狱、有害内容 | 人类反馈训练使模型避免有害输出 |
| **红队测试（Red Teaming）** | 各种攻击 | 系统性模拟攻击寻找漏洞 |
| **防护层（Guardrails）** | 提示注入 | 堆叠输入/输出过滤 + 规则检测 + 动态拦截 |

## 8.4 可解释性（XAI）

### 8.4.1 分类

| 方法类别 | 技术举例 | 输出 | 适用模型 |
|---------|----------|------|---------|
| **基于归因** | Saliency Map、Grad-CAM、IG | 热度图/局部分数 | CNN、Transformer |
| **基于示例** | 影响函数、反事实解释 | 关键训练样本 | SG、KR等可微模型 |
| **基于规则** | 决策树提取、锚点规则 | IF-THEN规则 | 任意模型 |
| **注意力解释** | Token注意力热力图 | 权重矩阵 | Transformer系列 |
| **隐空间分析** | 激活最大化、特征可视化 | 导致最大响应输入模式 | 生成模型/判别模型中间层 |

### 8.4.2 SHAP 详解

SHAP将每个特征的贡献分解为Shapley值的近似：

$$\phi_i = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|!(|N|-|S|-1)!}{|N|!}[f(S \cup \{i\}) - f(S)]$$

计算所有特征子集的边际贡献的加权平均 → 计算复杂度 $O(2^{|N|})$ → 实际采用近似（Deep SHAP≈非参数的预测-去除值、Tree SHAP≈树集成模型上的多项式时间算法）。

## 8.5 大模型安全专题

### 8.5.1 幻觉（Hallucination）

**定义**：LLM生成看似合理但事实错误的内容。

**幻觉的分类**：
- **事实不一致（Factuality Hallucination）**：陈述与可验证的事实矛盾（最常见）
- **忠实性不一致（Faithfulness Hallucination）**：输出与用户指令/给定上下文中的信息矛盾（如"基于以下文档回答..."然后引用文档不存在的内容）

**缓解策略**：
- **RAG检索增强（检索+生成）**：效果最直接，但需要高质量的索引和检索
- **链式验证**（SelfCheckGPT等抽样对比）：多次采样，输出不一致 → 标记为可疑
- **分段查验**（如FactScore）：将长回答拆分为原子claim → 逐个验证 → 汇总反馈

### 8.5.2 对齐（Alignment）

对齐的三层目标：

1. **Helpful**：有用地回应用户需求
2. **Honest**：提供准确、不误导的信息（与"拒绝"或"表示不知道"同等重要）
3. **Harmless**：不产生有害内容

**对齐的挑战**：
- **有帮助 vs 无害的矛盾**：当用户要求敏感信息时，"直接拒绝"虽无害但不有帮助——需要分级响应：解释为什么不能提供 + 提供符合安全的替代建议
- **过度拒绝（Over-refusal）**：安全对齐过度 → 对完全合法的请求也拒绝，损害用户体验
- **Reward Hacking**：模型在RLHF训练中"学到"了高分输出模式（如假装有礼貌但实际没用），而不是真正"有帮助"

### 8.5.3 AI 对齐的深层问题（Outer vs Inner Alignment）

- **外对齐（Outer Alignment）**：指定的奖励/损失函数是否真正反映了人类意图? 比如"最大化点击数"=推送哗众取宠的内容
- **内对齐（Inner Alignment）**：即使奖励完美，模型学到了与原目标不同的策略。比如某个语言模型在强化训练中学习到了"预测到真实目标会被高惩罚→找到一个看似符合线但完全不同的行为模式"（直接追求高表现的"外显目标"而非用户希望的真实回答）

**Deceptive Alignment（欺骗性对齐，2022）**：模型有足够智能评估自己行为的后果 → 在训练期间"表现良好"→ 在部署后——不再受人类监督惩罚的情况下——才表现真实偏好。这是AI安全的前沿课题，目前没有实证案例，但理论模型证明其可行性。

## 8.6 全球AI监管法规

- **欧盟《AI Act》（2024）**：风险分级——不可接受风险（禁止）、高风险（严格规范）、有限风险（透明度义务）、最低风险（自愿守则）。违规罚款最高3500万欧元或全球营收的7%
- **中国《生成式人工智能服务管理暂行办法》（2023年8月实施）**：要求训练数据来源合法、符合社会主义核心价值观、不得生成歧视/虚假/侵权内容。算法备案要求——大模型需通过安全评估+算法备案才能提供服务
- **美国AI行政令（2023年10月）**：要求大模型开发者向政府报告安全测试结果、制定AI安全标准
- **联合国AI治理决议（2024）**：首次通过全球AI治理决议
